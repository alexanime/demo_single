# demo_single

Демонстрационная версия `telegram-booking-bot` — публичный бот, на котором потенциальный клиент может потыкать кнопки и почувствовать, как такой бот будет выглядеть у его студии. **Реальные записи никуда не сохраняются**, мастеру ничего не приходит, никто не приедет.

База кода идентична `telegram-booking-bot`: репозиторий используется как «каркас» для будущих ботов. Демо-режим включается флагом `DEMO_MODE=1` и меняет поведение в нескольких точках:

- `/start` показывает краткое объяснение «это демо-бот» с ссылками на канал и контакт.
- В главном меню скрыт пункт **Админ** — клиенту он не нужен, а владельцу демо тоже ни к чему.
- В клиентском меню добавлена кнопка **«О тарифах 💼»** со ссылкой на канал.
- При выборе слота в календаре бот **не** пишет в БД и **не** уведомляет администратора. Вместо этого редактирует сообщение в подтверждение «Готово! Вы записаны на DD-MM-YYYY в HH:MM» с пометкой про демо и CTA «Хотите такого же бота — напишите @username».
- В разделе **«Мои записи»** показывается короткое объяснение, как этот раздел будет выглядеть у настоящего бота, без реальных записей.

Если поставить `DEMO_MODE=0` — бот работает ровно как `telegram-booking-bot` со всеми админскими режимами и БД-записями.

## Запуск

```bash
pip install -r requirements.txt
cp .env.example .env
# Отредактировать .env: вставить TELEGRAM_TOKEN, DEMO_USERNAME, DEMO_TARIFFS_URL
python bot.py
```

Для постоянной работы на VPS — поднять `bot.py` через `systemd` (пример на Ubuntu):

```ini
[Unit]
Description=demo single bot
After=network.target

[Service]
WorkingDirectory=/home/ubuntu/demo_single
ExecStart=/usr/bin/python3 /home/ubuntu/demo_single/bot.py
Restart=always
User=ubuntu

[Install]
WantedBy=multi-user.target
```

## Конфигурация

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `TELEGRAM_TOKEN` | — | Токен от `@BotFather`. **Должен быть отдельным**, не тот, что у боевого бота. |
| `DEMO_MODE` | `1` | `1`/`0` — включить/выключить демо-логику. |
| `DEMO_USERNAME` | `@al_username` | Юзернейм для CTA-кнопок. Куда писать желающим заказать. |
| `DEMO_TARIFFS_URL` | `https://t.me/your_channel` | Ссылка на канал с тарифами для кнопки «О тарифах». |
| `OWNER_ID` | `0` | Используется только при `DEMO_MODE=0`. |
| `BOT_DB_PATH` | `bot.db` | Используется только при `DEMO_MODE=0` (в демо БД создаётся для совместимости, но в неё ничего не пишется). |

## Что не реализовано в демо (намеренно)

- админ-меню и команды для мастера
- уведомления админу о новых записях
- БД-операции по бронированию слотов
- запрос контакта (телефон) у клиента — после выбора слота сразу подтверждение

Эти возможности есть в `telegram-booking-bot` и доступны при `DEMO_MODE=0`.
